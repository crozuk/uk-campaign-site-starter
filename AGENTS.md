# AGENTS.md

## Project purpose

This repo is a reusable Astro starter for UK advocacy and campaign websites. It is not a one-client site. Keep systems generic, keep demo content replaceable, and keep future editing easy for humans and coding agents.

## Stack

- Astro 5
- TypeScript
- Plain CSS
- Astro content collections
- Cloudflare Pages
- Cloudflare Pages Functions

## Important commands

```bash
npm install
npm run dev
npm run preview
npm run typecheck
npm run build
npm run build:template
npm run build:production
npm run deploy:live
npm run deploy:preview
npm run prepublish:check
npm run og:generate
```

## Where content lives

- Site-wide metadata, nav, socials, legal, visibility, MP lookup options: `src/data/site.ts`
- Homepage demo content blocks: `src/data/campaign.ts`
- CTA defaults and MP email templates: `src/data/actions.ts`
- Long-form campaign pages: `src/content/pages/*.md`
- Blog posts: `src/content/blog/*.md`
- FAQ entries: `src/content/faq/*.md`
- Quotes: `src/content/quotes/*.md`

## Where theme tokens live

- Theme source of truth: `src/data/theme.ts`
- Semantic CSS tokens: `src/styles/tokens.css`
- Base/global styling: `src/styles/global.css`
- Motion and reduced-motion rules: `src/styles/motion.css`
- Shared component patterns: `src/styles/components.css`

## Visibility toggle rules

- `src/data/site.ts` is the only place to control page and homepage-section visibility.
- `visibility.pages` controls which markdown pages are generated by `src/pages/[page].astro`.
- `visibility.blog.showInNavigation` controls blog nav visibility without removing blog code.
- `visibility.homeSections` controls homepage section rendering.
- `mpLookup.pageEnabled` and `mpLookup.showOnHomepage` control the MP lookup surfaces.
- Hidden sections should be removed from rendered output where practical. Do not hide them only with CSS.

## Accessibility expectations

- Keep semantic landmarks and heading order intact.
- Preserve visible focus states.
- Respect `prefers-reduced-motion`.
- Avoid hover-only interactions.
- Keep contrast strong and forms keyboard-accessible.

## Placeholder marker rules

The starter intentionally contains placeholder/demo markers that must remain easy to find until replaced. Current build checks look for markers like:

- `TODO_`
- `REPLACE_ME`
- `EXAMPLE_`
- `lorem ipsum`
- dummy example email addresses
- placeholder site URLs

Do not remove the production safeguard unless explicitly requested.

## Pre-publish requirements

- `npm run build` should keep working locally in preview mode.
- Production builds must use `PUBLISH_CHECK_MODE=production npm run build`.
- If you add new placeholder conventions, update `scripts/prepublish-check.mjs` and `src/data/publish-check.ts`.

## Reusable component rule

Do not hardcode campaign-specific copy, legal text, stats, or CTA labels inside reusable components unless the component is explicitly a demo-only content block. Prefer data files and content collections.

## Implementation note

The content pages use `src/pages/[page].astro` instead of separate route files so enabled pages are generated from content slugs and disabled pages are omitted from the build more cleanly.
