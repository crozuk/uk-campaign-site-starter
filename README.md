# UK Campaign Site Starter

Reusable Astro starter for UK advocacy, civic-tech, and campaign websites. It ships with realistic fictional demo content, a central site config, markdown blog support, a Cloudflare Pages Function for MP lookup, and a production safeguard that blocks placeholder content from reaching a live site.

## What this starter is for

This repo is meant to be reused across campaigns, not cloned as a one-off brochure site. The design, content structure, and docs are set up so a team can:

- replace demo copy without rewriting components
- switch whole pages and homepage sections on or off from central config
- keep longer campaign content in dedicated content files
- publish blog posts in markdown from day one
- deploy a mostly static site to Cloudflare Pages while keeping the MP lookup as a small edge function

## Stack

- Astro 5
- TypeScript
- Plain CSS with CSS custom properties
- Astro content collections for markdown content
- Cloudflare Pages for hosting
- Cloudflare Pages Functions for `/api/mp-lookup`
- Wrangler for local Pages preview and deployment compatibility

## Local development

Install dependencies:

```bash
npm install
```

Run the site locally:

```bash
npm run dev
```

That runs the static Astro dev server. For a local Pages-runtime smoke test after building:

```bash
npm run preview
```

Useful commands:

```bash
npm run typecheck
npm run build
npm run build:template
npm run build:production
npm run deploy:live
npm run deploy:preview
npm run prepublish:check
npm run og:generate
```

## Visibility toggles

Central visibility rules live in `src/data/site.ts`.

- `visibility.pages` controls which content-driven pages are generated by `src/pages/[page].astro`
- `visibility.blog.showInNavigation` hides or shows the blog in nav while keeping blog code available
- `visibility.homeSections` controls which homepage sections are rendered at build time
- `mpLookup.pageEnabled` and `mpLookup.showOnHomepage` control the MP lookup surfaces

This keeps hidden sections and content pages out of the rendered HTML rather than just hiding them with CSS.

## Blog support

The blog uses Astro content collections from day one. Add a post in `src/content/blog/` with frontmatter like:

```md
---
title: Example title
date: 2026-03-09
summary: Short summary text
tags:
  - tag-one
  - tag-two
coverStyle: mesh
---
```

Supported `coverStyle` values are `mesh`, `signal`, and `horizon`.

## MP lookup

The MP lookup page lives at `/contact-your-mp`, with a homepage teaser block using the same reusable component.

How it works:

1. The browser requests `/api/mp-lookup?postcode=...`
2. The Pages Function normalises the postcode
3. It looks up the parliamentary constituency via [postcodes.io](https://postcodes.io/)
4. It looks up the current MP via the [UK Parliament Members API](https://members-api.parliament.uk/)
5. It returns a normalised JSON payload for the frontend component

The frontend behaviour is config-driven:

- `enabled`
- `showOnHomepage`
- `pageEnabled`
- `showParty`
- `showProfileLink`
- `emailMode`
- `copyTemplateSource`
- `ctaStyle`

Default email templates live in `src/data/actions.ts`.

## Publish check

The repo includes a placeholder safeguard in `scripts/prepublish-check.mjs`.

- Development: placeholders allowed
- Preview: placeholders warn
- Production: placeholders fail the build

The checker scans `src/`, `public/`, `functions/`, `astro.config.mjs`, and `wrangler.toml` for markers such as:

- `TODO_`
- `REPLACE_ME`
- `EXAMPLE_`
- `lorem ipsum`
- dummy emails
- placeholder site URLs

Default `npm run build` uses preview mode so the starter still builds locally. For a real launch build:

```bash
PUBLISH_CHECK_MODE=production npm run build
```

For this template repo specifically, the manual Cloudflare deploy commands intentionally use development-mode checks so demo markers do not block your working preview:

```bash
npm run deploy:live
npm run deploy:preview
```

## Cloudflare Pages deployment

Recommended production setup:

- Framework preset: Astro
- Build command: `PUBLISH_CHECK_MODE=production npm run build`
- Build output directory: `dist`
- Node version: current LTS

This project is static-first. Only the `functions/api/mp-lookup.ts` route runs server-side.

See [`docs/DEPLOYMENT.md`](./docs/DEPLOYMENT.md) for the full Cloudflare Pages setup, preview notes, and Web Analytics guidance.

## Previews

Cloudflare Pages preview deployments work well with this starter because:

- the site builds statically
- placeholder markers warn instead of fail in preview mode
- Pages Functions ship alongside the static assets

For previews in the Cloudflare dashboard, use `PUBLISH_CHECK_MODE=preview` unless you want placeholder builds to fail there as well.
